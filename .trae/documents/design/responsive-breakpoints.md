# 响应式断点文档

## 设计理念
采用移动优先的响应式设计理念，确保在各种设备上都有良好的显示效果和用户体验。

## 响应式断点系统

### 1. 断点定义
| 断点名称 | 最小宽度 | 最大宽度 | 设备类型 | 用途 |
|---------|---------|---------|---------|------|
| sm | 640px | 767px | 平板 | 平板设备的基础布局 |
| md | 768px | 1023px | 平板/小型桌面 | 平板和小型桌面设备 |
| lg | 1024px | 1279px | 桌面 | 标准桌面设备 |
| xl | 1280px | 1535px | 大型桌面 | 大型桌面设备 |
| 2xl | 1536px | - | 超大型桌面 | 超大型桌面设备 |

### 2. 布局策略

#### 2.1 移动端布局 (< 640px)
- 单列布局
- 导航栏折叠为汉堡菜单
- 卡片列表单列显示
- 表单元素全屏宽度
- 按钮和交互元素增大尺寸，便于触摸操作

#### 2.2 平板布局 (640px - 1023px)
- 双列布局
- 导航栏显示部分菜单项
- 卡片列表双列显示
- 表单元素适当调整宽度

#### 2.3 桌面布局 (>= 1024px)
- 多列布局
- 完整导航栏显示
- 卡片列表三列或四列显示
- 侧边栏固定或可折叠
- 内容区域居中对齐，限制最大宽度

### 3. 组件响应式设计

#### 3.1 网格布局
```tsx
// 移动端单列，平板双列，桌面三列，大型桌面四列
<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-6">
  {/* 卡片内容 */}
</div>
```

#### 3.2 导航栏
```tsx
// 移动端隐藏，平板及以上显示
<nav className="hidden sm:flex">
  {/* 导航菜单 */}
</nav>

// 移动端显示，平板及以上隐藏
<button className="sm:hidden">
  {/* 汉堡菜单图标 */}
</button>
```

#### 3.3 文本大小
```tsx
// 不同屏幕尺寸下的文本大小
<h1 className="text-2xl sm:text-3xl lg:text-4xl">
  标题文本
</h1>
```

#### 3.4 间距调整
```tsx
// 不同屏幕尺寸下的间距
<div className="p-4 sm:p-6 lg:p-8">
  {/* 内容 */}
</div>
```

### 4. 设计注意事项

1. **移动优先**：先设计移动端布局，然后逐步扩展到更大的屏幕尺寸
2. **断点选择**：使用标准的断点值，确保与设计工具和其他框架兼容
3. **内容优先级**：在小屏幕上优先显示最重要的内容
4. **触摸友好**：确保交互元素有足够的尺寸，便于触摸操作
5. **性能优化**：根据屏幕尺寸加载适当大小的资源
6. **测试覆盖**：在各种设备和屏幕尺寸上测试布局和功能

### 5. 最佳实践

1. **使用相对单位**：使用 rem、em、vw 等相对单位，避免使用固定像素值
2. **弹性布局**：使用 flex 和 grid 布局，确保元素能够自适应屏幕尺寸
3. **媒体查询**：合理使用媒体查询，避免过度使用
4. **响应式图片**：使用 srcset 和 sizes 属性，提供不同尺寸的图片
5. **字体缩放**：确保字体大小在不同屏幕尺寸下都能良好显示
6. **内容可读性**：确保文本行长度在各种屏幕尺寸下都保持在最佳可读性范围内 (45-75 个字符)

### 6. 测试方法

1. **浏览器开发者工具**：使用 Chrome DevTools 或 Firefox DevTools 模拟不同设备尺寸
2. **实际设备测试**：在真实设备上测试，确保真实体验
3. **自动化测试**：使用自动化测试工具，确保响应式布局在各种屏幕尺寸下都能正常工作
4. **性能测试**：测试不同屏幕尺寸下的页面加载速度和性能
